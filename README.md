# 내PC클리너 (안전하게 파일 지우기)

고른 파일을 무작위 데이터로 여러 번 덮어쓴 뒤 삭제해, 일반 복구 프로그램으로는 되살릴 수 없게 만드는 Windows 유틸리티입니다. 설치 없이 실행되는 단일 `.exe`입니다.

일반적인 "삭제"나 "휴지통 비우기"는 파일의 실제 데이터를 지우지 않고 자리 표시만 지우기 때문에, 복구 도구로 되살릴 수 있습니다. 이 프로그램은 데이터 자체를 덮어써서 그 위험을 줄입니다.

제작: [MINARC](https://minarc.kr) · 공식 배포처: [Releases](../../releases)

## 다운로드

**[→ 최신 릴리즈에서 내려받기](../../releases/latest)**

| 파일 | 플랫폼 | 내용 |
|---|---|---|
| `mypccleaner.zip` | Windows | 실행파일 (zip, 다운로드 권장) |
| `mypccleaner.exe` | Windows | 실행파일 단독 |
| `mypccleaner-mac.zip` | macOS | 앱(.app) + 스크립트(.command) + 사용법 |

요구사항: Windows 10 / 11 (64비트, ARM 호환) 또는 macOS 10.13 이상.
두 버전 모두 설치가 필요 없습니다.

### macOS 버전

macOS 기본 기능(AppleScript · 셸)만 사용하는 스크립트 기반 앱으로, 파이썬 등 설치가 필요 없습니다. 서명되지 않은 앱이라 **첫 실행 시 `내PC클리너.app`을 우클릭 → [열기] → [열기]** 로 한 번 허용해야 합니다(이후 더블클릭). 자세한 내용은 `mac/사용법-맥.txt` 참고.

## 사용법

1. `mypccleaner.exe` 실행
2. **파일 고르기**로 파일 선택 (Ctrl/Shift로 다중 선택) 또는 **폴더 통째로**
3. 목록 확인 후 **지우기** → 확인

> ⚠️ 한 번 지운 파일은 복구할 수 없습니다. 삭제 전 목록을 반드시 확인하세요.

---

## 동작 방식

파일마다 다음 순서로 처리합니다.

1. 무작위 데이터로 **7회 덮어쓰기** (`BCryptGenRandom` 기반 난수)
2. 파일 크기를 0으로 만들기
3. 무작위 이름으로 변경
4. 최종 삭제

**폴더 통째로**를 선택하면 안의 모든 파일(하위 폴더 포함)을 위 방식으로 안전삭제한 뒤, 남은 빈 하위 폴더와 선택한 폴더 자체까지 제거합니다.

## 무결성 검증

배포 바이너리가 변조되지 않았는지 확인하세요. **Releases에서 받은 파일만 신뢰하세요.**

**v1.1.0 · `mypccleaner.exe`**

```
크기      167,886 bytes
SHA-256   e303f578f8d51455135e74fa45da867c7c4a73044e68f2e677cf7bdf858d8cd6
```

확인 방법 (PowerShell):

```powershell
Get-FileHash .\mypccleaner.exe -Algorithm SHA256
```

이 값과 다르면 실행하지 마세요.

### SmartScreen 경고

유료 코드 서명 인증서를 사용하지 않은 무료 유틸리티라, 첫 실행 시 "알 수 없는 게시자" 경고가 표시될 수 있습니다. 이는 악성코드라는 뜻이 아니라 **서명으로 제작자가 인증되지 않았다**는 의미입니다. 위 SHA-256으로 원본 여부를 직접 확인하거나, 아래 방법으로 소스에서 직접 빌드해 사용하세요.

## 직접 빌드하기

### Windows 실행파일 (C)

MinGW-w64 크로스 컴파일:

```bash
# 1) 아이콘 리소스 컴파일 (생략 가능 — 생략 시 기본 아이콘으로 동작)
x86_64-w64-mingw32-windres app.rc -O coff -o app_res.o

# 2) 본체 컴파일
x86_64-w64-mingw32-gcc src/secure_delete_win.c app_res.o -o mypccleaner.exe \
  -mwindows -O2 -municode \
  -lcomdlg32 -lshell32 -lole32 -lbcrypt -lgdi32 -lcomctl32
```

> 참고: 컴파일러 버전·최적화 옵션에 따라 산출물의 해시는 달라질 수 있습니다. 위 SHA-256은 배포 바이너리의 지문이며, "이 소스로 그 바이너리가 빌드되었음"을 증명하는 값은 아닙니다(재현 가능 빌드 아님).

### Python 버전 (Windows / macOS / Linux)

```bash
python src/secure_delete.py                      # GUI
python src/secure_delete.py a.txt b.jpg --yes    # CLI
```

## 프라이버시

이 프로그램은 다음을 하지 않습니다.

- 인터넷 통신 / 데이터 전송
- 개인정보 수집
- 백그라운드 상주 / 자동 실행 등록
- 다른 파일·프로그램 설치나 변경

사용자가 직접 고른 파일을 삭제하는 동작만 수행합니다. 소스코드로 확인할 수 있습니다.

## 한계 (반드시 읽어보세요)

**SSD·USB·SD카드 등 플래시 저장장치에서는 완전삭제가 보장되지 않습니다.**
웨어 레벨링(wear leveling)·오버프로비저닝 때문에 덮어쓰기가 실제 물리 블록에 도달하지 않을 수 있습니다. 이 도구뿐 아니라 모든 덮어쓰기 방식의 공통 한계입니다. 중요한 자료라면 다음을 함께 고려하세요.

- 저장장치 전체 암호화 후 키 폐기 (crypto erase) — 예: BitLocker
- 제조사 제공 Secure Erase / Sanitize 명령
- 물리적 파기

**덮어쓰기 횟수에 대해.**
이 도구는 7회 덮어쓰기를 사용하지만, 현대 자기디스크(HDD)에서는 **1회 덮어쓰기로 충분**하다는 것이 NIST SP 800-88(Media Sanitization) 등의 견해입니다. 다회 덮어쓰기는 과거 저밀도 매체를 전제로 한 관행에 가깝습니다.
